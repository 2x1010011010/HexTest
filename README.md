Тестовое задание для Redux Games
Потрачено времени:
Архитектура: (основные классы, стейт машина, загрузка ассетов) 6 часов
Игровая логика: 16 часов
UI: 4,5 часа
Редактор гексагональной карты: 6,5 часов

Игра стартует с Initial сцены, где находится точка входа, на сцену выкидывается префаб Bootstrapper, он создает экземпляр класса Game со стейт машиной игры.
Далее происходит регистрация сервисов, загрузка прогресса (сохранение прогресса не реализовано), загрузка сцены, выкидывание на сцену нужных префабов и прокидование необходимых зависимостей.
После стейт машина переходит в GameLoopState где уже начинается сама игра.
Сейчас для прокидывания и регистрации сервисов используется ServiceLocator Но его можно заменить на Zenject

Добавление новых карт в игру:
В игре реализован HexGridEditor (Tools -> Grid -> Hex Grid Editor)
1 При помощи которого создается префаб сетки.
Сетки сейчас можно выбрать двух типов: прямоугольная и круговая
Так же в редакторе можно выбрать один из типов для каждой клетки
На данный момент это только будет ли клетка включена, выключена и является ли клетка спавн точкой
Выключенные точки не включаются в префаб как объекты, на спавн точках в начале игры будут появляться стеки гексагонов
Данный редактор при необходимости может быть расширен другими видами клеток на сетке

2 Создается LevelConfig (Create -> Static Data -> Level Config)
В LevelConfig мы закидываем Material (CellMaterial), выбираем цвета для фишек на уровне, сюда же переносим созданный префаб сетки. 
Настройка камеры сейчас не используется нигде, её можно не трогать. 
Дальше задается условие выигрыша (сколько нужно убрать фишек с поля)

3 Созданный конфиг игры добавляется в LevelConfigsList

Сейчас при старте игры случайным образом выбирается один из конфигов в LevelConfigsList.

Известные проблемы:
1 Не корректный поворот фишек при анимации перелета (последствия не правильно выбранной модели гексагона, исправляется другой моделью и правкой метода расчета поворота)
2 При перетаскивании стека гексагонов на карту во время анимации перелета с таким же цветом как у гексагонов, может начаться анимация мерджа гексагонов до того как поставили стек на карту
3 При установке гексагона определенного цвета между двумя такими же иногда не верно считается последовательность действий и собираются не все гексагоны (один стек остается в стороне)
